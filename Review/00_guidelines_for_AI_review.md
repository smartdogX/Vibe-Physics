# Guidelines for AI-Assisted Review

AI-assisted review should be faster, more selective, and more retrieval-based than learning. The goal is not to build a topic from scratch, but to strengthen memory, sharpen distinctions, expose weak points, and prepare for exams or problem solving.

## 1. Treat review as compression and retrieval

During review, AI should focus on helping the student recall and reorganize material they have already seen. It should emphasize:

* key equations,
* core structures,
* common tricks,
* likely pitfalls,
* exam-style reasoning.

The emphasis should be on retrieval and reactivation, not long exposition.

## 2. Start by mapping what matters most

AI should begin review by identifying:

* the central results of the chapter,
* the most important formulas,
* the most likely exam questions,
* the distinctions most likely to be confused.

This gives the student a target rather than repeating the whole chapter.

## 3. Prefer questioning over explaining

In review mode, AI should ask more than it tells.

A good review rhythm is:

* ask the student to compute a small result,
* ask them to identify a symmetry,
* ask them to predict a sign,
* ask them to interpret a term,
* then give brief feedback.

This is because retrieval is one of the strongest forms of review.

## 4. Focus on high-yield structures

AI should emphasize the parts most worth remembering, such as:

* canonical definitions,
* standard Hamiltonians,
* basis choices,
* symmetry reductions,
* standard eigenstates,
* limiting cases,
* typical sign errors.

Review should not spend too much time on secondary details unless the student asks.

## 5. Use short derivation chains

When re-deriving something in review mode, AI should keep the chain compact. The purpose is not to teach every detail again, but to remind the student of the logic.

For example:

* define the basis,
* write the Hamiltonian,
* identify symmetry,
* reduce the block,
* read off the spectrum.

This keeps the student connected to the structure without slowing the review.

## 6. Prioritize error correction

Review should strongly focus on the types of mistakes that happen under exam pressure, such as:

* missing (\sqrt{n}) factors,
* wrong fermionic minus signs,
* forgetting a prefactor like (-J),
* confusing parity with exchange,
* mixing basis conventions,
* misidentifying which states get interaction energy.

AI should actively surface these failure points.

## 7. Compare nearby concepts quickly

In review mode, AI should help the student separate easily confused pairs in a compact way, for example:

* bosons vs fermions,
* commutator vs anticommutator,
* even vs odd parity,
* singlet vs triplet,
* site inversion vs particle exchange.

This comparison-based review is often more effective than isolated explanation.

## 8. Use exam-style prompts

AI review should sound closer to oral-exam or problem-session questioning, such as:

* “What is the matrix in this basis?”
* “Which states get (U)?”
* “Why does this state decouple?”
* “What symmetry reduces the Hamiltonian?”
* “What is the ground state when (U\gg J)?”

These prompts help the student practice recall in the same style they will need later.

## 9. Emphasize memory anchors

AI should help the student form compact anchors, such as:

* one canonical formula,
* one representative basis,
* one symmetry trick,
* one characteristic physical picture,
* one list of common mistakes.

Review should reduce the topic to a manageable set of durable memory points.

## 10. Use weak-point targeting

If the student shows uncertainty, AI should not restart the whole chapter. Instead, it should zoom in on the weak point only.

For example:

* review only fermionic signs,
* review only parity classification,
* review only interaction terms,
* review only the block diagonalization.

This keeps review efficient.

## 11. End with rapid self-test

A review session should end with a short set of questions the student should answer without help. For example:

* apply one operator to one state,
* write one Hamiltonian matrix,
* classify one state by symmetry,
* state one physical interpretation,
* identify one common mistake.

The point is to check readiness, not just comfort.

## 12. The role of the student

In AI-assisted review, the student should:

* try to answer from memory first,
* ask AI to test rather than just explain,
* use mistakes to identify weak spots,
* focus on retrieval speed and precision,
* ask for summaries only after active recall.

Review is successful when the student can reproduce the key structure quickly and correctly.
